## [Unreleased]
### Added
- **Track 092 (Feature)**: Implemented Real-Time Voice Conversations for Skylar Chat via the Gemini Live API (`gemini-3.1-flash-live-preview`). Developed a new hook `useSkylarLive.ts` that manages WebAudio context recording, mic capturing, PCM encoding, and model audio response playback. Integrated the live voice toggle into the Skylar Interaction Panel UI (`SkylarInteractionPanel.tsx`), properly loading stage configurations into the live session.
- **Track 091 (Bugfix)**: Fixed critical issue where a valid `VITE_GEMINI_API_KEY` was being falsely rejected with `API_KEY_INVALID` and an empty/dummy key fallback due to Genkit's `@genkit-ai/googleai` initialization timing. Updated `backend/services/genkitService.ts` to explicitly provide `apiKey: activeGeminiKey` into the Google AI plugin object during initialization rather than relying on it attempting to scan `process.env.GEMINI_API_KEY` at import-load time (which often resulted in evaluating as `undefined` before `dotenv` population in native module resolution).
- **Track 090 (Bugfix)**: Fixed Gemini Chat crash `[400 Bad Request] Please enable tool_config.include_server_side_tool_invocations...`. Removed `googleSearchRetrieval` config overriding from `genkitService.ts` prompts as Genkit and the native Node SDK currently have an unresolved friction point with tool schema mapping.

- **Track 089 (Bugfix)**: Fixed `FirebaseError: [code=permission-denied]` exceptions crashing the app by ensuring all `onSnapshot` listeners correctly capture errors using `handleFirestoreError`.
- **Track 088 (Bugfix)**: Fixed critical "Journey stage not found" fallback in `configService.ts` ensuring Skylar successfully gracefully defaults to `DEFAULT_JOURNEY_STAGES` if the Firestore collection relies on unseeded states. Addressed `ERR_MODULE_NOT_FOUND` within backend Genkit execution during stage loads by removing local imports of `firebaseAdmin`.
- **Track 087 (Phase 3)**: Deepened the WavVault Implementation. Added background intel extraction mechanism (`analyzeWavvaultArtifactFlow`) that actively summarizes and categorizes User-Skylar interactions via `/api/wavvault/chat`. Refactored `DistilledArtifact` typing to enforce realistic schemas (e.g., `extractedSkills`, `industryRelevance`). Implemented automated document parsing intelligence and visualization via the new `WavvaultIntelHub` tab in the User Dashboard, enabling the user to search and visualize their discrete knowledge nodes as a secure intelligence hub.
- **Track 087 (Phase 2)**: Added "No-Code Maintenance for Phases". Migrated `defaultStageContent` logic into a new Firestore collection (`journeyPhaseConfigs`). Expanded `AgentOps.tsx` with a tabbed interface ("Global Settings", "Phase Management", "Genkit Traces") to allow non-technical team members to edit and deploy Stage settings out-of-band without full code deployments. Updated `backend/services/genkitService.ts` to dynamically fetch this data, using the node environment instead of relying on statically encoded configurations.
- **Track 087 (Phase 1)**: Harmonized Dev and Production environments for Genkit processing. Ensured `@genkit-ai/googleai` and `@genkit-ai/vertexai` share the exact identical schemas and prompt definitions. Added a comprehensive `API_KEYS.md` file tracking exact setup configuration for developers.
- **Track 086**: Add Google Maps Tool. Created a custom `search_google_maps` tool in `backend/services/genkitService.ts` and registered it with `allTools`. Added `GOOGLE_MAPS_API_KEY` to `.env.example` and `TECH_SPECS.md`. Includes mock response data if the API key is not configured across environments.
- **Track 085**: Add Google Search Grounding to Genkit. Modified `backend/services/genkitService.ts` to remove the mocked `searchWebTool` and natively injected `googleSearchRetrieval: {}` into the `ai.generate` and `skylarPrompt` generator configurations. Determined Google Maps Grounding is natively implicit to the Search Grounding index and requires a custom Maps API integration for deep geographic functionality.
- **Track 083**: Fix Skylar Chat `API_KEY_INVALID` Crash in Production. Resolved an issue where an invalid user-provided `VITE_GEMINI_API_KEY` (e.g., revoked, expired, or malformed) would cause Genkit to throw a `[400 Bad Request] API key not valid` error and crash the fallback flow since the active key check still erroneously forced the `googleai/gemini-3-flash-preview` model default. Implemented strict error parsing to intercept `API_KEY_INVALID` and cleanly fall back natively to Google Cloud Vertex AI ADC (`vertexai/gemini-1.5-flash`), bypassing the standard API key plugin logic if standard authentication fundamentally fails.
- **Track 082**: Fix Skylar Chat `[VertexAI.ClientError] 404 Not Found` Error in Production. Resolved an issue where the Genkit `@genkit-ai/vertexai` plugin threw a 404 error because the auto-generated Firebase project (`gen-lang-client-...`) lacks the Vertex AI API natively. Modified `genkitService.ts` initialization to decouple the explicit `FIREBASE_PROJECT_ID` override so Vertex AI correctly utilizes natively injected ADC discovery during production. Additionally, reconfigured genkit fallback logic to explicitly pass the dynamically resolved model to prompt generation, preventing static closure evaluation loops from locking the primary `model` argument to broken defaults.
- **Track 081**: Fix Genkit `FAILED_PRECONDITION` error on Cloud Run production deployments missing `VITE_GEMINI_API_KEY` secrets. Migrated the backend `genkitService` to dynamically use the `@genkit-ai/vertexai` plugin with `vertexai/gemini-1.5-flash` relying on native Application Default Credentials (ADC) as a fallback if the standard API key cannot be successfully extracted during initialization.
- **Track 080**: Fix Skylar Chat `FAILED_PRECONDITION` Error in Production. Resolved an issue where Genkit threw a `FAILED_PRECONDITION: Please pass in the API key` when running in Cloud Run/Preview. Genkit's `googleAI` initialization was being invoked at module-load time before environment variables could be definitively resolved across platforms, resulting in an `undefined` config. Forced `process.env.GEMINI_API_KEY` to be universally populated within `getGeminiApiKey()` and explicitly invoked it prior to executing `runJourneyStageFlow` inside the `/api/skylar/chat-journey` Express endpoint to guarantee Genkit natively detects the key upon generating a response.
- **Track 079**: Fix Skylar Chat Flow Failure ("Failed to chat with Skylar" error). The UI was previously genericizing all non-200 responses. Updated `skylarService.ts` to surface raw JSON proxy or server error messages to the user interface. Rewrote the internal logic in `genkitService.ts` to prevent Genkit from executing client-side `fetch()` relative path endpoints when invoking internal skylar tool functions, fetching database context locally via `@firebase/firestore` native server initialization instead.
- **Track 078**: Fix Skylar Chat Crashing the Backend (`The backend server is currently starting up`). Refactored dynamic imports inside `server.ts` route handlers (`runJourneyStageFlow`, `genkitTracer`, `vertexDiscoveryService`) to be static top-level ES6 imports, preventing `ERR_MODULE_NOT_FOUND` exceptions during run-time that resulted in the node process crashing and restarting when `SkylarService` made POST requests to `/api/skylar/chat-journey`. This prevents the AI Studio proxy from intercepting the 500-state with its fallback `Starting Server...</title>` HTML page.
- **Track 077**: Fix Application Configuration Missing Error on Cloud Run. Updated `src/main.tsx` to fallback to `firebase-applet-config.json` imported variables for `firebaseApiKey` and `firebaseProjectId` if the `VITE_FIREBASE_*` environment variables are undefined (as they are in Cloud Run directly after an export from GitHub without manual secret injection).
- **Track 076**: Implemented dynamic runtime environment variable injection for production deployment. Fixed an issue where ESBuild would statically compile out `import.meta.env` references to `<undefined>` during Cloud Run deployments (as secrets are unavailable at `npm run build` time). Implemented a `window.__ENV__` secure payload inside Express server HTML requests, successfully mirroring Vite's `import.meta.env` behavior on the frontend.
- **Track 075**: Fix Cloud Run Deployment Failure (`API key does not start with "SG."`). Wrapped `sgMail.setApiKey()` initialization inside a try-catch block and added a strict `startsWith("SG.")` check to prevent invalid placeholder keys from bubbling an unhandled exception during Node server startup.
- **Track 074**: Fix `Uncaught TypeError: Cannot read properties of undefined (reading 'VITE_FIREBASE_API_KEY')`. Fully replaced `import.meta.env` references within the unified pipeline by statically defining an inclusive `import.meta.env` object in the ESBuild plugin mapping algorithm, mapping all standard `MODE`, `DEV`, `PROD` context switches over from `process.env`.
- **Track 074**: Removed Vite and transitioned to a unified `esbuild` + `tailwindcss/cli` build pipeline.
