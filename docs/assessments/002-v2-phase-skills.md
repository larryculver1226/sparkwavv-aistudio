# Assessment: Phase-Specific Skills V2

## Overview
While the current suite of phase-specific tools (Tracks 114-118) provides excellent structural waypoints for the user, there are still a few UX gaps where Skylar could reduce friction, especially regarding data entry, macro-market awareness, and real-time practice.

## Gap Analysis & Proposed Tool Plan

### Phase 1: Dive-In (Onboarding & Triage)
*   **Gap**: Users hate filling out forms or answering endless onboarding questions about their history.
*   **Proposed Tool (`parseResumeToVault`)**: Allows the user to paste their existing resume text, which Skylar autonomously shreds and categorizes into the foundational 'WavVault' taxonomy (Work History, Skills, Education) instantly, bypassing generic Q&A.

### Phase 2: Ignition (Deep Self-Reflection & DNA)
*   **Gap**: Core values and energy are tracked, but we lack data on the user's *environmental* and *operational* constraints.
*   **Proposed Tool (`assessOperatingStyle`)**: Categorizes the user's optimal working environment (Maker vs. Manager schedule, Async vs. Sync, Remote vs. Office) through conversational probing, saving it to their Career Blueprint to heavily filter future job matches.

### Phase 3: Discovery (Market Alignment)
*   **Gap**: Titles and pivots are tracked, but what about macro industry health? (e.g., "Is UX Research dying?")
*   **Proposed Tool (`analyzeIndustryTrends`)**: Aggregates macro trends for the user's specific sub-industry to provide a "Market Heatmap" widget, answering questions about automation risk, hiring volume, and geographic hotspots.

### Phase 4: Branding (Narrative & Assets)
*   **Gap**: Resumes and portfolios are handled, but users struggle with verbal networking.
*   **Proposed Tool (`draftElevatorPitch`)**: Distills the complex `career_blueprint` into a contextual, 30-second conversational script (or a 2-sentence LinkedIn bio) designed strictly for human-to-human networking events rather than ATS parsing.

### Phase 5: Outreach (Connection & Execution)
*   **Gap**: Negotiation and tracking are in place, but live interview performance anxiety remains a major blocker.
*   **Proposed Tool (`triggerMockInterview`)**: Sets up a constrained temporal mode where Skylar strictly role-plays a specific hiring manager avatar (e.g., "Meta Product Lead") for 5 conversational turns, then outputs a structured feedback scorecard widget on their performance.

## Next Steps
This plan outlines a "V2" suite of phase-specific skills designed to round out the user experience from initial ingestion to final interview prep. Please review these proposed tools and provide your approval or adjustments before we begin the Setup & Build phases.
